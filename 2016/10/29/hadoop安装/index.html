hadoop安装

发表于 2016-10-29   |   分类于 hadoop学习

目的

写本片博客主要是记录自己安装过程中的坑，但是别人没有注意到的一些点。

  • linux 中不同用户的操作不会共享，例如root用户的操作，不会共享给普通用户
  • 用ls -l命令查看文件夹拥有者
  • 同步时钟时候出现的一些问题
  • 安装过程中出现任意bug可以从hadoop-(version)/logs/文件夹下查看

需要的材料

作者的操作系统是win7 64位。

1.在本教程前，需要下载并且安装好VMware虚拟机。
2.下载Centos 操作系统镜像(本文使用的是Centos7)
3.下载hadoop-(version).tar.gz(本文使用的是hadoop-2.5.2)
4.下载SSH Secure File Transfer Client 用于文件传输

1.虚拟机中安装Centos

1. 安装流程

  • 选择典型安装(这里的典型安装是系统给出的默认配置，能够适合大多数人)

  • 选择需要安装的操作系统路径（本文的是Centos）

2. 克隆HadoopSlave（完全克隆）

克隆前需要关闭虚拟机，启动或者挂起均无法克隆

2.配置SSH远程连接

1.启动SSH服务

在Centos 中直接执行SSH，如果找不到命令，则需要安装SSH

2.配置免密钥登录

为什么需要配置免密钥登录呢？
原因是当Hadoop master 启动时候，这时候要求跟Slave进行通信，在这个阶段如果不配置免密钥通信，则每次启动都需要输入多次密码

下面我们看看如何配置

1.操作前提是启动了两个虚拟机，1个是master，1个是slave

2.在master 上执行ssh-keygen -t rsa ,然后一直点击enter确认即可。

3.copy master 上的密钥到slave 上，这时候即可达到免密钥通信效果

4.执行 ssh slave ，如果没有要求输入密码，则配置成功

3.Linux 环境配置

1.配置同步时钟

自动配置同步时钟
该项操作需要同时在master ,slave 节点配置

使用Linux命令配置

crontab -e

该命令是vi编辑命令，执行后会进入编辑器界面，vi 用法: 按i进入插入模式，按Esc，软后键入：wq保存退出。
输入i，进入插入模式,键入下面一行代码,(星号之间和前后都有空格)

0 1 * /usr/sbin/ntpdate cn.pool.net.org


上面这行代码中，cn.pool.net.org 是一个国际时间服务器。整句话的意思是 每天凌晨1点执行一次时间同步

手动配置同步时钟
直接在Terminal中运行下面命令

/usr/sbin/ntpdate cn.pool.ntp.org

但是主编遇到一些小问题，配置的时候同步出来的时间没有奏效，突然想到肯能是自己的系统时区配置有问题

系统时区配置
命令行中输入

tzselect



接下来跟着提示走之后，会弹出下面信息，然后就没有了，之前也是一脸茫然，不过仔细阅读之后，发现，
这些步骤并没有修改Linux中的任何东西， TZ=’Asia/Shanghai’; export TZ

扩展 配置同时配置多台主机同步时钟

3.jdk 的安装

直接执行yum install java 即可

4.关闭防火墙

执行如下命令即可

systemctl stop firewalld.service

3.传输hadoop压缩文件，并且配置hadoop

1.执行解压命令

tar -zxvf hadoop-(version).tar.gz

2.配置环境变量 hadoop-env.sh

进入hadoop解压目录，目录下有 etc 文件夹，etc下有hadoop文件夹，该文件夹下面的均是配置文件
本文的hadoop解压在了/home/yilinfeng 下 ，这里的yilinfeng 是操作系统用户的目录，每个用户拥有自己对应的目录

vi /home/yilinfeng/hadoop-2.5.2/etc/hadoop/hadoop-env.sh


在文件的靠前部分找到下面的一行代码：

export JAVA_HOME=${JAVA_HOME}


将其修改为

export JAVA_HOME=/usr


需要注意的是这里的JAVA_HOME，如果是通过yum 下载的java 的话，目录是/usr ，如果是自己下载的话，目录需要自己指定java jdk 目录

这里还有一点需要注意的是，如果在~/.bash_profile 中配置了JAVA_HOME ,在这里的话，可以直接引用为

export JAVA_HOME=$JAVA_HOME


~/.bash_profile中配置JAVA_HOME方法，

export JAVA_HOME=/usr;


并且执行

source ~/.bash_profile


使配置文件生效

3.配置环境变量yarn-env.sh

该配置文件步骤跟上面相似

vi /home/yilinfeng/hadoop-2.5.2/etc/hadoop/yarn-env.sh


找到

#export JAVA_HOME=/home/y/libexec/jdk1.6.0/


将其前面的#号去掉即可

#export JAVA_HOME=/usr


这里的话，估计将其配置为

export JAVA_HOME=$JAVA_HOME


亦可实现同样的效果。

4.启动hadoop集群

1.配置Hadoop 启动的系统环境变量

该配置需要同时在两个节点上操作，master slave ，命令如下：

vi ~/.bash_profile


将下面环境变量添加在.bash_profile末尾:

#hadoop
export HADOOP_HOME=/home/yilinfeng/hadoop-2.5.2
export PATH=$HADOOP_HOME/bin:$HADOOP_HOME/sbin:$PATH


然后执行如下命令，是配置文件不用重启就能生效

source ~/.bash_profile

2.创建数据目录

该操作需要在两个节点上同时进行操作，master slaves

在yilinfeng 的用户住目录下面，创建临时数据目录，命令如下:

mkdir /home/yilinfeng/hadoopdata

3.启动Hadoop集群

1.格式化文件系统

格式化命令如下，该操作仅需要在master节点上执行:

hdfs namenode -format


如果控制台显示的信息不包含Exception/Error ，则表示操作成功，若出现问题，可以到/home/yilinfeng/hadoop-2.5.2/logs 下查看

2.启动Hadoop

使用start-all.sh 启动Hadoop集群

cd ~/hadoop-2.5.2
sbin/start-all.sh

4.检测进程是否启动

通过 yum install 安装的java 没有jps程序，所以无法通过jps命令查看
但是若存在jps命令，终端执行jps

jps


如果master出现

  • ResourceManager
  • NameNode
  • Jps
  • SecondaryNameNode

这四个程序，则表示启动成功，其中Jps属于java进程，但是与hadoop无太大关系，jps命令主要是查看系统中的java进程。
slave中执行jps,可以看到

  • DataNode
  • NodeManager
  • Jps

5.web ui 查看集群是否启动成功

访问http://master:50070/，检查namenode 和datanode 是否正常。UI页面展示如下

访问http://master:18088/ 检查Yarn是否启动正常。

小结

操作中，所有操作均在普通用户下进行，切勿在root用户下操作一点，普通用户下操作一点，这样子的话，会导致Permission Deny问题，需要提权的操作，可以通过sudo 命令实现。
