前端无限滚动实现

发表于 2016-10-30   |   分类于 javaScript

最近开发的照片墙，由于用到的界面排版是瀑布流，所以自然会牵扯到无限滚动这门技术。
瀑布流的话，优缺点如下:

优点

  • 能够很好的把最新数据呈现出来，体现数据的时效性。
  • 用户体验上，用户无需点击下一页即可加载更多数据。对应照片型应用，用户体验非常好。

缺点

  • 用户无法跳到指定页面(当信息量更新很快，用户如果需要看之前的信息需要刷新很多次，而且无法跳页加载)
  • 一直滚动的话，一直加载动态元素，加大浏览器的负担，内存消耗严重。

适用场景:

  • 适用于信息更新快的应用，但错过某条信息无所谓，例如微博，能够很好的把最新数据展现出来
  • 页面元素少，在加载大量数据下，也不会太影响应用响应性能

总的来说，在图片应用上使用瀑布流还是不错的。
